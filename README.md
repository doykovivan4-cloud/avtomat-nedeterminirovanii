# Практическая работа: Конечный автомат Недетерминированный 

## 1. Тип конечного автомата

### Конечный недетерминированный автомат (НКА) — математическая модель вычислений, обобщающая детерминированные конечные автоматы (ДКА).
Его ключевая особенность: для одного и того же входного символа из текущего состояния может быть несколько возможных переходов (или даже ни одного).

### Виды конечного автомата:
| Вид конечного автомата | Краткое описание | Особенности |
|---------------------|----------------|-------------|
| Детерминированный конечный автомат (ДКА / DFA) | Каждый переход однозначно определяется текущим состоянием и входным символом | Из любого состояния для каждого символа — не более одного перехода; нет ε‑переходов |
| Недетерминированный конечный автомат (НКА / NFA) | Может иметь несколько возможных переходов из одного состояния для одного и того же входного символа | Возможны множественные переходы; может содержать ε‑переходы (переходы без чтения символа) |
| НКА с ε‑переходами (ε‑НКА / ε‑NFA) | Разновидность НКА, допускающая переходы по пустому символу (ε) | Переходы могут выполняться без чтения входного символа; формально расширяет возможности НКА |
| Минимизированный ДКА | ДКА с минимально возможным числом состояний, эквивалентный исходному автомату | Эквивалентен исходному ДКА по распознаваемому языку; все недостижимые и эквивалентные состояния устранены |
| Автомат с выходом (Мура / Moore) | Конечный автомат, генерирующий выходные сигналы | Выход зависит только от текущего состояния (в автомате Мура) |
| Автомат с выходом (Мили / Mealy) | Конечный автомат с выходами, привязанными к переходам | Выход зависит от текущего состояния и входного символа (в автомате Мили) |
| Вероятностный конечный автомат | Переходы между состояниями происходят с определёнными вероятностями | Каждому переходу сопоставлена вероятность; используется в моделях с неопределённостью |
| Квантовый конечный автомат | Теоретическая модель, основанная на принципах квантовой механики | Использует квантовые состояния и суперпозицию; обобщает классические автоматы |

## 5. Проверить работу автомата 
| Входная последовательность | Пошаговое прохождение | Результат | Пояснение |
|------------------------|--------------------|-----------|----------|
| `ab` | 1. $q_0 \xrightarrow{a} q_1$<br>2. $q_1 \xrightarrow{b} q_2$ | ✅ **Принято** | Завершается в принимающем состоянии $q_2$. |
| `aab` | 1. $q_0 \xrightarrow{a} q_1$<br>2. $q_1 \xrightarrow{a} q_1$<br>3. $q_1 \xrightarrow{b} q_2$ | ✅ **Принято** | После чтения `b` попадает в $q_2$. |
| `abb` | 1. $q_0 \xrightarrow{a} q_1$<br>2. $q_1 \xrightarrow{b} q_1$<br>3. $q_1 \xrightarrow{b} q_2$ | ✅ **Принято** | На последнем `b` переходит в $q_2$. Возможен переход на шаге 2 сразу в $q_2$, но это не влияет на результат. |
| `aaabbb` | 1. $q_0 \xrightarrow{a} q_1$<br>2. $q_1 \xrightarrow{a} q_1$<br>3. $q_1 \xrightarrow{a} q_1$<br>4. $q_1 \xrightarrow{b} q_1$<br>5. $q_1 \xrightarrow{b} q_1$<br>6. $q_1 \xrightarrow{b} q_2$ | ✅ **Принято** | Последний `b` переводит в $q_2$, несмотря на длинную среднюю часть. |
| `ba` | 1. $q_0 \xrightarrow{b}$ → **нет перехода** | ❌ **Отвергнуто** | Первый символ `b`, а из $q_0$ переход есть только по `a`. |
| `a` | 1. $q_0 \xrightarrow{a} q_1$<br>Больше символов нет | ❌ **Отвергнуто** | Заканчивается в $q_1$, а не в $q_2$. Не удовлетворяет условию «заканчивается на `b`». |
| `aba` | 1. $q_0 \xrightarrow{a} q_1$<br>2. $q_1 \xrightarrow{b} q_2$<br>3. $q_2 \xrightarrow{a}$ → **нет перехода** | ❌ **Отвергнуто** | После попадания в $q_2$ (на `b`) следующий символ `a` не имеет перехода. |
| `b` | 1. $q_0 \xrightarrow{b}$ → **нет перехода** | ❌ **Отвергнуто** | Первый символ — `b`, из $q_0$ нет перехода по `b`. |
| `aaa` | 1. $q_0 \xrightarrow{a} q_1$<br>2. $q_1 \xrightarrow{a} q_1$<br>3. $q_1 \xrightarrow{a} q_1$<br>Больше символов нет | ❌ **Отвергнуто** | Заканчивается в $q_1$. Нет `b` в конце. |
| пустая строка | Начинаем в $q_0$, символов нет | ❌ **Отвергнуто** | Не начинается с `a`, не попадает в $q_2$. |
